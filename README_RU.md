# CODE13 ACCESS GRID — статическая версия HTML/CSS/JS

## Что это за версия

Проект полностью работает без PHP и без базы данных. Его можно:

- открыть локально двойным кликом по `index.html`;
- разместить на GitHub Pages;
- разместить на любом обычном статическом хостинге.

## Локальный запуск

1. Распакуйте архив.
2. Откройте `index.html` в браузере.
3. Админ-панель открывается через `forge/index.html`.

## Как пользоваться админкой

1. Откройте `/forge/`.
2. Войдите.
3. Добавьте, измените, скройте или переставьте ссылки.
4. Нажмите **СКАЧАТЬ LINKS.JS**.
5. Замените скачанным файлом файл `data/links.js` в проекте.
6. Загрузите изменение в GitHub — после публикации новые ссылки увидят все посетители.

До замены `data/links.js` изменения являются только локальным черновиком в текущем браузере.

## GitHub Pages

1. Создайте репозиторий.
2. Загрузите содержимое папки проекта в корень репозитория.
3. Откройте `Settings → Pages`.
4. В разделе `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку `main` и папку `/ (root)`.
6. Сохраните настройки.

Главная страница будет доступна по адресу вида:

`https://username.github.io/repository/`

Админка:

`https://username.github.io/repository/forge/`

## Важное ограничение статического сайта

Чистый HTML/CSS/JS не может безопасно записывать файлы обратно в GitHub и изменять сайт для всех посетителей. Поэтому панель формирует новый `links.js`, а публикация выполняется заменой файла в репозитории.

Защита логином в статической версии скрывает интерфейс от случайного доступа, но не является серверной авторизацией. При этом посторонний человек всё равно не сможет изменить опубликованный сайт без доступа к вашему GitHub-аккаунту или хостингу.
